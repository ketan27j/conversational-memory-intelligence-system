# 2026-07-17 — M2 closed: hybrid search, ranking, and a real ranking bug caught by independent verification

## What shipped

Hybrid search (`implementation/retrieval/`): vector similarity + keyword search (`ts_rank_cd`,
normalized) + entity match combine into one `semantic` signal (C7), blended with
recency/frequency/importance into the final rank (C6). A relevance floor
(`RELEVANCE_FLOOR = 0.20`) gates each candidate individually so the system can honestly abstain
("I don't know") instead of guessing (INV-6). A token-budget cap is enforced (INV-3).
`write_gate/pipeline.py` was extended to call `retrieval.indexer.index_memory(...)` at write time
— closing the gap found the day before (2026-07-16) where `embedding`/`memory_entity` were never
actually populated. `api/main.py` gained `POST /v1/memories:retrieve` per `api_contracts.md`.

Demo: `pytest implementation/tests/test_retrieval.py -v` → 11/11 passed (35/35 full suite at the
time), mypy clean, ruff clean.

## A real bug, found by independent verification — not by the build itself

L4 VERIFY's first pass (a separate model instance, fresh context) **rejected** the milestone. The
bug: the relevance-floor abstention check was gated on the *single top-by-final-score* candidate,
not on each candidate's own semantic score. That's a real problem — a recent, important, but
completely irrelevant memory could out-rank a genuinely relevant answer on the blended score,
which would then cause the whole query to fail the relevance bar and abstain even though a correct
answer existed in the store. In other words: a distractor could suppress a real answer by winning
a ranking contest it wasn't even relevant to.

The fix: gate abstention per-candidate on `semantic` score, not on whichever candidate happens to
rank first overall. A regression test was added specifically for this shape of failure
(`test_relevant_answer_survives_an_unrelated_recent_important_distractor` — see
`implementation/tests/test_retrieval.py`). L4 VERIFY's second pass approved.

## Why this one matters for the retrospective

This is the clearest example in the whole project of independent verification catching something
the build's own test suite (green, 11/11 at the time of the first L4 pass) did not — because the
bug wasn't "does the code crash," it was "does the *design* handle an adversarial input shape,"
which requires someone (or something) actively trying to break the abstention logic rather than
just running the happy-path tests that were written alongside the code that has the bug in the
first place. Same-author tests share the same blind spot as the code they're testing; a
genuinely separate reviewer does not.
