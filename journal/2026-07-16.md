# 2026-07-16 — M1 closed out; M2 begun

## M1's quiz-me gate answered

Before M1 could be marked done, three questions were answered by hand (not by an AI reviewer) to
confirm actual understanding, not just a passing test suite:

1. Why the bare-token secret-detection heuristics use the specific thresholds they do (24+ char
   mixed-case runs, 32+ char hex runs), and why a 40-character git commit SHA getting redacted as
   a false positive is an *accepted* trade-off rather than a bug — over-redacting a harmless hash
   is low-severity; under-blocking a real secret is the direction the threat model prioritizes
   avoiding.
2. What happens differently when a pasted message is *entirely* a secret versus a secret embedded
   mid-sentence (422 rejection + nothing stored, vs. redact-and-store with `pii_blocked: true`),
   and the exact 0.8 span-coverage threshold that decides which branch fires.
3. Why a naive "any bare 12-digit number" regex to catch Aadhaar numbers would have been a bad
   idea — it would break the false-positive tests (ordinary phone numbers, order IDs) and violate
   the threat model's explicit "both directions" principle: a regex with no credential *shape*
   signal trades away "don't cry wolf" to gain recall on one national-ID format.

M1 was marked done the same day this was answered — 2026-07-16, two days after the build and
AI-verifier approval themselves (see `2026-07-15.md`'s note on that gap).

## M2 begun: hybrid search and ranking

G0 pre-flight for M2 (search and ranking) started the same day. Two things worth recording from
that pre-flight itself, before any code was written:

- **A gap found beyond the obvious.** `write_gate/pipeline.py::process_turn` (M1) inserts into
  `memory` but never populates the `embedding` column or the `memory_entity` table — M1 built
  fact extraction and the write gate, not indexing. Hybrid search would be meaningless without
  write-time indexing, so M2's scope had to expand to include wiring that in, not just building a
  read-time scorer. This is exactly the kind of scope decision the handbook wants surfaced
  explicitly rather than silently absorbed.
- The ranking formula itself (`0.4·similarity + 0.2·recency + 0.2·frequency + 0.2·importance`)
  was already specified in `reconstruction/first_principles.md`'s C6 — this milestone's job was
  to implement and validate it, not invent it from scratch.
