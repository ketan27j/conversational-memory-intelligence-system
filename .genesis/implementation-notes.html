Decisions that constrain new work

  • ADR-003 — one PostgreSQL database, no separate vector/graph store for v1 → see design/ADR-003-one-postgres-database.md
  • ADR-004 — tenant isolation enforced inside the database (RLS), application filters are a second layer only, never the thing relied on → see design/ADR-004-separation-inside-the-database.md
  • ADR-005 — secrets checked before anything else, feeds M1 → see design/ADR-005-check-secrets-first.md
  • Hardening note (surfaced during M0's L4 VERIFY quiz): the cmis table-owner role is a Postgres superuser (default behavior of the official Docker image) and unconditionally bypasses RLS regardless of FORCE ROW LEVEL SECURITY — nothing beyond one-time schema setup should ever connect as cmis. Not yet enforced by anything beyond convention/documentation here.
