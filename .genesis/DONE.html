1 · The Locked Spec (what we are building)

Cognitive job: Stop losing information between conversations — selectively. Decide whether a fact is worth keeping (write gate), store it as a typed record (event/fact/preference), retrieve it via hybrid search (vector + keyword + entity) ranked on similarity/recency/frequency/importance, resolve contradictions (newer+more-confident wins, history kept), and forget on a schedule (archive, never hard-delete except on explicit user request).

Inputs: authenticated user's conversation turns; explicit correction ("that's wrong") and deletion requests  →  Outputs: retrieved memory context injected into the assistant's prompt (typed, cited, relevance-gated so it can say "I don't know"); write-gate accept/reject decisions, logged

Autonomy level: Autonomous by default — the write gate and nightly forgetting job act without human approval. The human touchpoints substitute for a review step (correction, deletion) rather than gating every write.  |  Human-in-the-loop points: user can correct a stale/wrong memory; user can request deletion (must clear reads instantly, search index within 60s)

Failure tolerance: Must degrade to slow-but-correct, never fast-but-wrong — if the memory store is down or slow, the lookup times out and the assistant answers without memory rather than failing the turn. Memory is an enhancement, never a single point of failure.  |  Trust boundary: The end user is untrusted (can type hostile text); user identity comes only from the login token, never the request body; row-level isolation enforced inside the database itself (not application code); retrieved memories are handed to the model labelled as data, never as instructions (prompt-injection-via-memory mitigation).

2 · Definition of Done (binary gates — checked by the VERIFIER, not the maker)

Architecture style chosen with justification; service graph exists; boundary types stated; no cycles in dependency graph (Phase 1 gate — largely satisfied by design/ADR-001..007)
Service modules defined; at least one state machine drawn with explicit states; every outbound call has a timeout strategy (Phase 3 gate)
Memory tiers separated explicitly (event/fact/preference/working); RAG pipeline uses hybrid retrieval; every response cites its source; freshness strategy defined (Phase 6 gate)
Threat model exists with specific adversary categories; prompt injection mitigated (memories labelled as data, never instructions); RBAC/RLS implemented; audit trails append-only (Phase 11 gate — threat_model.md already written)
Cross-user leak test: zero leaks, down from 0.92 baseline (hard gate, M0)
Secrets findable in store: zero, down from 1.00 baseline; passes in both directions (real keys blocked, innocent phrases not blocked) (hard gate, M1)
Outdated preference wins < 0.33 of the time, aiming near-zero (hard gate, M3)
Deleted memory gone from reads instantly, gone from search within 60s (hard gate, M4)
Full Deliverable-3 benchmark re-run and reported side by side against the naive baseline (hard gate, M5)
mypy implementation/ clean AND pytest implementation/tests/ green in CI
One verify pass by a separate agent/model (L4)
No context-graph.json invariant violated by the change (INV-1 tenant isolation, INV-2 no secrets stored, INV-4 deletion is real)
