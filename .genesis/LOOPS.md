# LOOPS.md — How conversational-memory-intelligence-system Gets Built

This file is **dev-only**. It is HOW we build conversational-memory-intelligence-system across many sessions with a cheap
driver model. The loops are NOT product features. They are the harness that prompts the agent so
you don't have to.

> Generated by the **Genesis kit**. Agent-agnostic: every "load skill X" / "spawn checker" below
> resolves to a concrete invocation via `AGENT-ADAPTERS.md` (Hermes · Claude Code · Codex · any).

## G0 — Existence Pre-Flight (RUN BEFORE ANY MILESTONE STARTS)

Before writing one line of code, every milestone MUST answer: "is this already built, partially
built, or unbuilt?" The wiki and `implementation-notes.html` exist precisely to prevent re-building.
Skipping G0 = wasted tokens and a duplicate implementation that drifts.

**G0 procedure (5-minute fixed cost per milestone):**

1. **Read** `.genesis/wiki/index.md` — pick candidate pages by name match against the milestone's nouns.
2. **Drill** into 2–4 candidate pages. Look for: existing files, endpoints, types, decisions that constrain this work.
3. **Search** `.genesis/implementation-notes.html` for the milestone's nouns. It is the rolling source-of-truth for what's live (wiki can lag).
4. **Grep** the codebase for the function/route/symbol names the plan proposes — confirm they don't already exist. Search, don't assume.
5. **Write** a `## G0 Existence Pre-Flight` block at the top of the milestone's checkpoint file:
   ```markdown
   ## G0 · {{TIMESTAMP}}
   - wiki pages read: [[...]], [[...]]
   - implementation-notes searched for: "<noun>", "<noun>" — found/none
   - codebase grep: `<symbol>` — N hits (unbuilt / existing-will-reuse)
   - decisions that bind us: <decision-id>
   - verdict: UNBUILT | PARTIAL | BUILT
   ```

**G0 verdict outcomes:**
- `UNBUILT` → proceed to L1 BUILD.
- `PARTIAL` → revise milestone scope to "extend X" not "build X." Update `CURRENT.md`.
- `BUILT` → halt the milestone. Mark done. Surface the existing artifact's location.

**G0 is gate zero.** Same enforcement as G1–G5: no checkpoint G0 block = the milestone didn't start.

## Operating Mode

- **Driver model:** cheap by default (claude-sonnet-5). Flagship (claude-opus-4-8) only for ARCH decisions or hard debugging hops. Loops assume the driver is NOT a flagship.
- **Escalation:** driver may request the flagship for ONE hard sub-task, then return to cheap immediately.
- **Skill canon — ALWAYS loaded every loop, every session:**
  - `agentic-swe-master` (orchestrator — non-negotiable, routes everything)
  - `coding-orchestrator` (route before any code)
  - `modular-architecture` (boundary calls)
  - `production-readiness` (whenever touching backend)
- **`coding-orchestrator` explained:** the router skill is a lightweight classifier that reads the
  current task and picks the right domain skill (e.g. `coding-orchestrator` reads the task type
  and routes to `tdd`, `security-engineering`, `llmops-ai-agents`, etc.). If you don't have one,
  use `coding-orchestrator` from the agentic-swe-kit — it ships with genesis.
- **Skill resolution order:** agent-global skills → ~/Desktop/skills-directory → .genesis/skills/
  (see `AGENT-ADAPTERS.md` for per-agent paths).
- **Cognitive skills available** (from skills-directory — install separately if missing):
  `detective` (debug), `verify` (check output), `blueprint` (design before build),
  `scout` (explore options), `council`/`mirror` (adversarial self-review),
  `ghost` (shadow/canary), `foresight` (predict failures).
- Repo conventions live in `AGENTS.md`/`CLAUDE.md` and `.genesis/wiki/` — every loop reads them at start.

## Anti-Slop Spine — The 5 Gates

Every iteration of every loop passes through these gates IN ORDER. Failing any gate halts the
iteration. Failing the same gate three iterations in a row halts the loop and surfaces to the user.

| Gate | Question | Pass criterion | Fail action |
|---|---|---|---|
| **G1 Skill** | Did we load `agentic-swe-master` + router + the loop's required skill? Did we log `skills considered: […]; chose: X`? | Log line in checkpoint | Halt iter, force load, retry |
| **G2 Progress** | Did this iter measurably move the milestone forward? (file diff, test added, error count down, knowledge filed) | ≥1 quantified delta | Strike. 3 strikes → halt loop |
| **G3 Cost** | Tokens + tool calls under per-loop budget? | Under cap | Halt loop, surface |
| **G4 Quality** | `{{TYPECHECK_CMD}}` clean AND `{{TEST_CMD}}` green AND lint clean for files touched? | All checks pass | Spawn L2 DEBUG, retry |
| **G5 Verify** | Independent checker (different model OR fresh context) approves the artifact against the milestone spec? | APPROVE verdict + reasoning | Spawn L2 DEBUG with feedback, retry once |

Gates are **computed, not narrated** — run the command, paste the exit code. No checkpoint block = the iteration didn't happen.

## Checkpoint Protocol (markdown, not a database)

```
.genesis/checkpoints/
  CURRENT.md                  ← which loop is active (always overwritten)
  <milestone-id>.md           ← per-milestone, append-only iteration log
  debug-<date>-NNN.md         ← per-debug-session
  research-<topic>.md         ← per-research-target
```

**CURRENT.md** (always overwritten):
```markdown
# CURRENT
- active_loop: BUILD
- target: {{MILESTONE_ID}}
- iteration: 3
- last_gate: G4 (passed)
- last_action: <command + result>
- next_action: <the very next concrete step>
- model: claude-sonnet-5
- tokens_used: 18420
- tokens_budget: 50000
- skills_loaded: [agentic-swe-master, coding-orchestrator, ...]
```

**Per-target checkpoint** (append-only, one block per iter):
```markdown
## iter 3 · {{TIMESTAMP}}
- gate G1: pass · skills considered [...]; chose X
- gate G2: pass · +84 LOC <file>, +6 tests, -0 type errors
- gate G3: pass · 4.1k this iter, 18.4k cumulative
- gate G4: pass · typecheck clean, tests 6/6, lint clean
- gate G5: deferred (run at exit only)
- decision: <one-line rationale>
- next: <next step>
```

Resuming sessions read `CURRENT.md`, then the active per-target file, then append.

## The Five Loops

```
L1 BUILD     ── ships one milestone
L2 DEBUG     ── sub-loop, fires on test/typecheck failure        (skill: detective / systematic-debugging)
L3 RESEARCH  ── sub-loop, fires on knowledge gap                 (skill: scout / deep-research; writes to wiki)
L4 VERIFY    ── gate-loop, wraps every BUILD exit                (skill: verify / requesting-code-review)
L5 HEALTH    ── periodic; wiki-lint + skill-drift, reports only  (never auto-fixes)
```

L4 is never top-level. Build calls Debug + Research as needed. Build exits through Verify.

## L1 — BUILD LOOP

**Purpose:** complete one milestone end-to-end with a passing demo command.

**Inputs:** milestone id · success criteria (from PLAN.md milestone table) · demo command (the exact invocation that proves it works).

**Required skills (load before iter 1):** canon + `blueprint`/`writing-plans` + `test-driven-development` + milestone-specific (see table).

**Pseudocode:**
```
load_skills(canon + milestone_specific)
read(AGENTS.md, PLAN.md, wiki/index.md, milestone.success_criteria)
checkpoint_init(milestone_id)

plan = produce_micro_plan()            # files to touch, tests to add, demo cmd
write_to_checkpoint(plan)

iter = 0
while not milestone_done() and iter < 10:
    iter += 1
    G1_skill_gate()
    phase = next_phase(plan)
    if phase.needs_research: spawn(L3_RESEARCH, scope=phase.unknowns)
    edit_files(phase)
    run_tests(phase.test_target); run_typecheck(phase.workspace)
    G2_progress_gate(iter); G3_cost_gate(iter); G4_quality_gate(phase)
    if G4_failed: spawn(L2_DEBUG, scope=last_failure); continue
    checkpoint_append(iter, gate_results)

spawn(L4_VERIFY, artifact=demo_command_output)
if verify == REJECT: spawn(L2_DEBUG, scope=verify_feedback); retry_once()
mark_milestone_done(); update_CURRENT(); append_progress_to_PLAN_and_notes()
```

**Stop conditions:** success (demo passes + L4 APPROVE) · convergence failure (3 G2 strikes) · budget (50000 tokens OR 10 iterations).

## L2 — DEBUG LOOP

**Purpose:** root-cause one specific failure. Always a sub-loop.

**Required skills:** `agentic-swe-master`, `detective` (or `systematic-debugging`).

**Pseudocode:**
```
load_skills([agentic-swe-master, detective])
checkpoint_init(debug_id, parent=parent_checkpoint)
repro = build_minimal_reproduction()
if not repro: halt("cannot reproduce — surface")
hypotheses = list_hypotheses(repro)        # at least 3, from disjoint evidence
iter = 0
while not root_caused() and iter < 5:
    iter += 1; G1_skill_gate()
    h = pick_next_hypothesis()
    r = test_hypothesis(h)                 # MUTE intervention: disable the line, re-run
    checkpoint_append(iter, h, r); G2_progress_gate(iter); G3_cost_gate(iter)
    if r.confirms: root_cause = h; break
fix = apply_fix(root_cause); reg = write_regression_test(repro)
G4_quality_gate(fix + reg)
return {root_cause, fix, reg} to parent
```

**Stop conditions:** root cause + regression test, OR 5 iterations, OR 20000 tokens.
**Loop-breaker:** never run two consecutive blind fixes. Isolate (mute/git-diff/state-check) before editing.

## L3 — RESEARCH LOOP

**Purpose:** close a specific knowledge gap and write the answer to the wiki so the next loop has it.

**Required skills:** `agentic-swe-master`, `scout`, web-browse skill (only when web sources needed).

**Pseudocode:**
```
load_skills([agentic-swe-master, scout])
checkpoint_init(research_id)
sources = gather_sources(topic)            # repo files + specs + schema + web
iter = 0
while knowledge_gap_open() and iter < 5:
    iter += 1; G1_skill_gate()
    src = sources.next(); summary = read_and_summarize(src)
    write_wiki_page("wiki/concepts/<slug>.md", summary)
    update_index_md(); append_log_md()
    G2_progress_gate(iter); G3_cost_gate(iter); checkpoint_append(iter, src, pages_touched)
    knowledge_gap = re_eval_gap()
return wiki_pages_created to parent
```

**Stop conditions:** gap closed (parent confirms) OR 5 iters OR 20000 tokens OR 2 consecutive sources add zero new info.

## L4 — VERIFY LOOP

**Purpose:** keep maker away from checker. Single-shot. Always wraps a BUILD exit.

**Setup:**
- **Different model from maker.** (e.g. maker=claude-sonnet-5, checker=claude-opus-4-8 for the hard/e2e milestones; a cheaper checker for trivial ones.)
- **Fresh context.** Checker sees ONLY: goal + success criteria + artifact (diff or command output). NOT the build trail.
- **Context-graph aware:** checker is also handed the affected `context-graph.json` invariants and asked "does this change violate any downstream invariant?"

**Required skills:** `agentic-swe-master`, `verify` (or `requesting-code-review`).

**Pseudocode:**
```
load_skills([agentic-swe-master, verify])
context = { goal, spec_excerpt, artifact, invariants_at_risk }   # NO build_trail
verdict = checker_model.judge(context)     # APPROVE | REJECT | UNCERTAIN
checkpoint_append(verdict, reasoning)
APPROVE → PASS; REJECT → FAIL_WITH(reasoning); UNCERTAIN → surface_to_user
```

**Quiz-me gate (runs on APPROVE before milestone is marked done):**
The verifier generates 3 targeted questions from the artifact — one per category:
  1. Design decision: "Why was [interface/approach X] chosen over the obvious alternative?"
  2. Edge case:       "What happens when [specific boundary condition] occurs?"
  3. Change impact:   "What does [this migration / function / schema change] do?"
Post the questions to the human. Wait for answers. Log Q+A to the milestone checkpoint.
If the human cannot answer → downgrade verdict to UNCERTAIN, surface to user.
The milestone is NOT marked done until the Q+A block is written to the checkpoint.
```
quiz = verifier.generate_questions(artifact, 3)   # design · edge case · change impact
answers = wait_for_human(quiz)
checkpoint_append("Q+A block", quiz, answers)
if any(answer == "don't know" or answer == "skip") → verdict = UNCERTAIN → surface_to_user
APPROVE + Q+A block logged → mark milestone done
```

**Iteration cap:** 1. **Token budget:** 10000.

---

## L5 — HEALTH LOOP

**Purpose:** keep `wiki/`, `PLAN.md`, `decisions/` clean as build progresses. On demand or scheduled.
Reports findings only — **never auto-fixes.**
Skills: `agentic-swe-master` + any of: `learn` (gstack), `dogfood`, or manual wiki review.
(`wiki-lint` is not a standard skill — if your agent has it, use it; otherwise do a manual
pass: read every wiki page, flag stale entries, surface gaps. Write findings to `wiki/log.md`.)

## What the Driver Model is NOT Allowed to Do

- Skip G0 or G1. Ever.
- Mark a milestone done without L4 VERIFY APPROVE.
- Run more than one loop level deep without checkpointing the parent.
- Edit `DONE.html` or `PLAN.md` without explicit user request.
- Invent gate results — gates are computed (run the command, paste the exit code), not narrated.
- Continue past iteration cap or budget without surfacing.
- Touch files outside the per-milestone freeze boundary ({{FREEZE_BOUNDARY}}).
